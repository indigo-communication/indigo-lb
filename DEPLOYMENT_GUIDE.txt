## üîß .htaccess Configuration

### Current Settings:

‚úÖ **HTTPS Redirect:** Forces all HTTP traffic to HTTPS  
‚úÖ **Clean URLs:** Removes .html extensions (`/about-indigo` instead of `/about-indigo.html`)  
‚úÖ **Non-www:** Forces `indigo-lb.com` (not `www.indigo-lb.com`)  
‚úÖ **Security Headers:** XSS protection, clickjacking prevention  
‚úÖ **Compression:** Gzip for faster loading  
‚úÖ **Browser Caching:** 1 year for images, 1 month for CSS/JS  

### To Use www Instead:

Edit `.htaccess` lines 17-25:

```apache
# Comment out non-www section (lines 20-21):
# RewriteCond %{HTTP_HOST} ^www\.(.+)$ [NC]
# RewriteRule ^(.*)$ https://%1/$1 [R=301,L]

# Uncomment www section (lines 17-18):
RewriteCond %{HTTP_HOST} !^www\. [NC]
RewriteRule ^(.*)$ https://www.%{HTTP_HOST}/$1 [R=301,L]
```

## üêõ Troubleshooting

### Contact Form Not Sending Emails

**Possible Issues:**
1. Wrong email address in `contact.php` (line 44)
2. PHP `mail()` function disabled on server
3. Emails going to spam folder

**Solutions:**
1. Double-check email address in `contact.php`
2. Check spam/junk folder
3. Ask Namecheap support to enable PHP mail() function
4. Consider using PHPMailer or SMTP (contact developer)

### Images Not Loading

**Possible Issues:**
1. Incorrect file permissions
2. Case-sensitive file paths (Linux hosting)

**Solutions:**
1. Set folder permissions to 755, files to 644
2. Verify image paths match exact filename case

### Pages Show 404 Error

**Possible Issues:**
1. Clean URLs not working (.htaccess not loaded)
2. mod_rewrite not enabled

**Solutions:**
1. Verify `.htaccess` file uploaded correctly
2. Ask Namecheap to enable mod_rewrite
3. Access pages with `.html` extension until fixed

### HTTPS Not Working

**Possible Issues:**
1. SSL certificate not activated yet
2. Mixed content (HTTP resources on HTTPS page)

**Solutions:**
1. Wait 24 hours for SSL activation
2. Check for http:// links in HTML (should be https:// or protocol-relative)
